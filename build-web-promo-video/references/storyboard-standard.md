# Web promo storyboard standard

Use this reference before selecting scenes or writing narration.

## Establish one promise

Write one sentence that states what the viewer should understand after the video. Choose scenes that
prove that promise. Avoid a complete feature inventory.

## Recommended sequence

1. **Hook**: show a recognizable result or strongest visual state quickly.
2. **Core interaction**: demonstrate the action that defines the product.
3. **Depth**: show two or three distinct systems, states, or use cases.
4. **Proof**: show real scale, workflow, output, or validation when relevant.
5. **Return**: revisit the strongest visual with more context.
6. **CTA**: show the project name and real destination without unsupported claims.

## Timing

- Keep individual browser scenes roughly 2-8 seconds unless an interaction needs longer.
- Leave enough time for a viewer to read critical UI once.
- Match narration to the scene currently visible; do not describe the next product while the old
  scene remains on screen.
- Use fewer, stronger scenes rather than a rapid list of visually identical screens.
- Treat landscape and vertical crops as different compositions even when they share the timeline.

## Evidence

- Capture real project states, not reconstructed promotional mockups.
- Identify debug or seeded states in project facts when used for deterministic recording.
- Do not convert configuration limits into measured runtime claims.
- Verify every repository URL, count, benchmark, and version appearing in narration or subtitles.

## Narration

- Write for speech, not documentation: short sentences, concrete verbs, limited nested clauses.
- Keep text comfortably within each segment; synthesize and measure before final timing when using
  TTS.
- Avoid adjectives that are not visible or evidenced.
- Keep the CTA specific and restrained.
